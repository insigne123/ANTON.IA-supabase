# Apple-Inspired Methodology

Esta es la metodologia general de UI/UX para toda la app.

## Principio rector

La interfaz debe priorizar:

- simpleza antes que espectacularidad
- claridad antes que densidad
- facilidad de uso antes que cantidad de elementos
- foco en la tarea antes que autoexplicacion excesiva

Si un bloque no ayuda al usuario a entender, decidir o actuar, probablemente sobra.

## Reglas base

### 1. Menos elementos, mejor jerarquia

- evitar paneles redundantes
- evitar texto que explique decisiones internas de diseño o producto
- preferir un titulo fuerte y un subtitulo claro antes que muchos bloques explicativos

### 2. El producto habla en lenguaje de usuario

- no usar copy tipo "sistema visual", "criterio UX", "siguiente capa"
- no mostrar pensamiento interno del equipo como parte de la UI
- cada texto debe responder a una pregunta real del usuario: que puedo hacer aqui, que gano, que hago despues

### 3. Superficies silenciosas

- menos cards si no aportan una accion o una lectura real
- fondos sutiles, bordes precisos, sombras discretas
- evitar mezclar demasiados acentos visuales en una sola vista

### 4. Un camino evidente

- una accion principal por zona
- secundarios subordinados visualmente
- la pantalla debe sentirse obvia sin tutorial

### 5. Modo dia y noche con la misma filosofia

- dark mode no es invertir colores, es preservar claridad con menos ruido
- mantener separacion limpia entre fondo, panel y contenido
- el contraste debe ayudar a leer, no llamar la atencion porque si

## Anti-patrones

- heroes demasiado explicativos
- paneles de contexto duplicados
- cards de marketing dentro de herramientas de trabajo
- badges decorativos sin valor funcional
- ruido visual para justificar diseño

## Antes de implementar una UI

Preguntar siempre:

- esto ayuda al usuario o solo llena espacio?
- este texto suena a producto real o a nota interna del equipo?
- puedo resolverlo con menos bloques?
- la accion principal se entiende en menos de 3 segundos?

## Referencias a priorizar

- Apple HIG para claridad, estructura y sentido comun de producto
- Unsection para heroes minimalistas con buen ritmo visual
- Figcomponents para settings, dashboards y componentes silenciosos
