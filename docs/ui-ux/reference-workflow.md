# Reference Workflow

Este flujo se aplica antes de tocar cualquier UI importante.

## 1. Clasificar el problema

Elegir primero que tipo de problema es:

- layout
- componente
- flujo
- sistema visual
- accesibilidad

## 2. Buscar referencias correctas

- `Unsection` si el problema es composicion, hero o estructura general
- `Figcomponents` si el problema es un componente concreto o panel de dashboard
- `ScreensDesign` si el problema es secuencia, experiencia o flujo de producto
- `Toolfolio` si hace falta descubrir recursos, librerias o tooling extra

## 3. Resumir antes de implementar

Antes de codificar, dejar claro:

- que referencias se tomaron
- que patrones se rescatan
- que no se va a copiar literalmente
- como se adaptan al lenguaje visual del proyecto
- como se reducira ruido visual en vez de agregar elementos innecesarios

## 4. Implementar

- adaptar, no clonar
- respetar modo dia y noche
- mantener consistencia con componentes existentes
- mejorar estados, spacing y jerarquia
- simplificar primero; agregar elementos solo si mejoran comprension o uso

## 5. Auditar

Revisar como minimo:

- jerarquia visual
- responsive
- dark mode
- focus y estados
- claridad de CTA principal
- densidad visual
- cantidad de elementos realmente necesaria
