# Privacy legal basis matrix

Objetivo:
Tener una primera matriz simple de bases legales tentativas por tratamiento.

Que significa cada base en simple:
- Ejecucion del servicio: el tratamiento es necesario para prestar la funcionalidad que el usuario contrato o activo.
- Interes legitimo: existe una finalidad razonable, pero hay que justificar que no pasa por encima de los derechos de la persona.
- Cumplimiento operativo: se mantiene informacion necesaria para respetar bajas, seguridad o trazabilidad.

## Matriz inicial

### 1. Registro, login y sesiones
- Base tentativa: ejecucion del servicio.
- Efecto: permite autenticar usuarios, proteger acceso y operar la cuenta.

### 2. Organizaciones, roles y permisos
- Base tentativa: ejecucion del servicio.
- Efecto: permite separar workspaces y controlar acceso interno.

### 3. Gmail y Outlook conectados por el usuario
- Base tentativa: ejecucion del servicio.
- Efecto: permite usar integraciones pedidas por el propio usuario.

### 4. Busqueda de leads B2B
- Base tentativa: interes legitimo.
- Efecto: permite detectar prospectos profesionales para outreach comercial.
- Nota: requiere validacion, minimizacion y transparencia.

### 5. Enrichment de datos profesionales
- Base tentativa: interes legitimo.
- Efecto: mejora datos de contacto o contexto para el trabajo comercial.
- Nota: mientras mas invasivo sea el enrichment, mayor justificacion se necesita.

### 6. Guardado de leads, notas y estados CRM
- Base tentativa: ejecucion del servicio para el cliente; en datos de terceros, interes legitimo del uso B2B.
- Efecto: permite organizar pipeline y continuidad comercial.

### 7. Envio de correos
- Base tentativa: ejecucion del servicio para el usuario que usa la plataforma.
- Efecto: permite materializar el outreach desde canales conectados.
- Nota: el tratamiento del destinatario final debe convivir con la base del uso comercial y su derecho de oposicion.

### 8. Tracking de apertura, click y reply
- Base tentativa: interes legitimo y operacion del servicio.
- Efecto: medir efectividad, entregabilidad y respuesta.
- Nota: requiere transparencia especial porque es de las funciones mas sensibles.

### 9. Bajas, listas de exclusiones y no contactar
- Base tentativa: cumplimiento operativo del servicio y respeto de oposicion comercial.
- Efecto: evita nuevos envios a quien pidio no ser contactado.

### 10. IA, scoring y recomendaciones
- Base tentativa: ejecucion del servicio y/o interes legitimo, dependiendo de la funcion.
- Efecto: prioriza leads, sugiere acciones y apoya redaccion.
- Nota: exige mayor cuidado por perfilamiento y automatizacion.

### 11. Logs, seguridad y auditoria
- Base tentativa: interes legitimo de seguridad, soporte y defensa del servicio.
- Efecto: permite investigar errores, abusos e incidentes.

## Conclusiones simples

1. Consentimiento no parece ser la base principal del producto.
2. La zona mas delicada es interes legitimo en prospeccion, enrichment, tracking y scoring.
3. Las bases tentativas sirven para ordenar el proyecto, pero no reemplazan validacion legal final.
